# Reyna demo assets

This folder holds the visuals embedded in the main README.

## Files

- **`demo.mp4`** — main demo video (autoplayed + looped at the top of the README). **Drop your own screen recording here.**
- **`demo.gif`** — fallback animated GIF if you don't have a video. Generated by running `vhs demo.tape`.
- **`demo-poster.png`** — still image shown before the video loads (recommended: a single frame from the recording).
- **`demo.tape`** — [vhs](https://github.com/charmbracelet/vhs) script that regenerates `demo.gif` from a fake terminal walkthrough. Useful as a placeholder until you record a real WhatsApp-side demo.

## How to make the demo video

### Option A — real WhatsApp recording (best)

1. Open WhatsApp on your phone or WhatsApp Web on a second monitor.
2. macOS: ⌘ + Shift + 5 → screen recording → record the chat window.
3. Show:
   - A friend DMing the bot: `reyna find srikar's python notes`
   - Bot replying with a list + Drive link
   - Friend replying `summarize module 5`
   - Bot answering with the summary
   - Friend asking the same question in Hinglish: `aur asaan shabdon mein batao`
   - Bot answering in Hinglish
4. Save as `docs/demo.mp4`.
5. Optionally extract the first frame as `docs/demo-poster.png`:
   ```bash
   ffmpeg -ss 00:00:00 -i docs/demo.mp4 -frames:v 1 -q:v 2 docs/demo-poster.png
   ```

### Option B — terminal-based GIF (placeholder)

1. Install [vhs](https://github.com/charmbracelet/vhs):
   ```bash
   brew install vhs
   ```
2. Run from the repo root:
   ```bash
   vhs docs/demo.tape
   ```
3. This produces `docs/demo.gif`. To use it instead of the video, edit the README and uncomment the GIF line, comment out the `<video>` block.

### Option C — dashboard recording

If WhatsApp recording is fiddly, record the dashboard's **Search → Notes Q&A** flow:
1. Type a question in the Q&A tab.
2. Wait for the answer.
3. Click a follow-up chip ("in simpler words" / "give an example").
4. Show another follow-up turn appearing.
5. Save as `docs/demo.mp4`.

The video tag in the main README will pick it up automatically — no other changes needed.
